# [[Prompt Engineering]]

## 定义
通过设计和优化提示词（Prompt），引导大语言模型生成高质量输出的技术与方法。

---

## 核心要点

### 1. 基础技巧
- **清晰具体**: 明确任务目标和输出格式
- **角色扮演**: 赋予模型特定角色（专家、助手）
- **示例驱动**: 提供输入输出示例（Few-shot）
- **结构化提示**: 使用分隔符、列表、表格

### 2. 高级技巧
- **思维链 (CoT)**: 引导模型逐步推理
- **思维树 (ToT)**: 探索多条推理路径
- **自我反思**: 让模型评估自己的输出
- **迭代优化**: 多轮对话逐步完善

### 3. 常见模式
- **RAG 模式**: 检索 + 生成
- **Agent 模式**: 规划 + 执行
- **工具调用模式**: 感知 + 决策 + 执行
- **多轮对话模式**: 上下文 + 迭代

## 实践案例

### 案例 1: 代码生成提示
```markdown
# 角色
你是一位资深软件工程师，精通 Python 和 AI 开发。

# 任务
为以下需求生成代码：
[需求描述]

# 要求
1. 使用 Python 3.10+ 语法
2. 添加类型注解
3. 包含单元测试
4. 遵循 PEP 8 规范

# 输出格式
```python
# 代码实现
```

# 测试
```python
# 单元测试
```
```

### 案例 2: 文档总结提示
```markdown
# 角色
你是一位技术文档专家。

# 任务
总结以下文档的核心要点：
[文档内容]

# 要求
1. 提取 3-5 个关键点
2. 每个要点 1-2 句话
3. 使用 Markdown 列表格式
4. 添加 TL;DR 摘要

# 输出格式
## TL;DR
[一句话总结]

## 核心要点
- 要点 1
- 要点 2
- 要点 3
```

## 提示词模板库

### 1. 角色扮演模板
```markdown
你是一位 [角色]，拥有 [技能/经验]。

你的任务是 [任务描述]。

请按照以下格式输出：
[输出格式]
```

### 2. 思维链模板
```markdown
让我们一步步思考这个问题：

1. 首先，[第一步]
2. 然后，[第二步]
3. 接着，[第三步]
4. 最后，[结论]

请详细说明每一步的推理过程。
```

### 3. Few-shot 模板
```markdown
这里有一些示例：

示例 1:
输入: [输入 1]
输出: [输出 1]

示例 2:
输入: [输入 2]
输出: [输出 2]

现在请处理：
输入: [新输入]
输出:
```

## 💡 关键洞察

**从字幕中提取的核心观点**:

1. **提示词是编程**: 自然语言成为新的编程语言
2. **结构化重要**: 清晰的结构比复杂的语言更有效
3. **示例驱动**: Few-shot > Zero-shot
4. **迭代优化**: 一次到位很难，需要多轮完善
5. **模型差异**: 不同模型需要不同的提示策略
